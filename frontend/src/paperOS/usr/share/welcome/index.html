welcome to paperOS V.0.2!

paperOS is built ENTIRELY with javascript and javascript alone. Develop apps with the intuitive POK Kernel, have the flexibility of a cat at your fingertips; and really make your operating system yours.

What is new?

1. Upgraded file system!

We are now using something called OPFS (Origin Private File System). This allows us to pretty much have our own little section of your disk for reading and writing files to! In turn, this allows for near native speeds for reading and writing, along with a bunch of other fun things such as actually being able to see how much storage you hae left. We have also updated how the whole backend system works, going from old window.pok.fileSystem (a method used all the way back in posh2), to a more modern class-based system. These changes allow developers to work even faster, and users to use even faster. With this change, you can see major speed boosts in doing simple things such as reading and writing to files!

Complete application rework!

Creating applications has never been easier! Simply create a new file, extend the Application class, and bam! you got an application. Creating windows is much more trivial nowadays, because it's all class based and separated away from the Application workflow! Whenever you used to make an application, it would open up a new window, and that window was part of your application. But now the window isapart of the application, it no longer IS the application. A good example of how applications work is by heading over to /usr/share/welcome/main.js. That is what is creating this window!

POSH revamp!

Long before time had a name, POSH was the thing that displayed the text, requested the user input, sent the commands; it did it all. But what if you no longer wanted to use POSH as your main shell? You would have to re-create all the calls for it, all of the input for it, the way it was displayed, everything. But now, we have separated it into 2 parts. POT, which stands for PaperOs Terminal and the tride and true POSH, now ACTUALLY a shell. As a result, you can mix and match with your own software, from switching POT with something else to making your own fork of POSH
